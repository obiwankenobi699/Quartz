## Pattern Used: Two Pointer (Inward Comparison)

We compare characters from both ends and move inward.


## Implementation

```java
class Solution {
    public boolean isPalindrome(String s) {

        int start = 0;
        int end = s.length() - 1;

        while (start < end) {

            while (start < end && 
                   !Character.isLetterOrDigit(s.charAt(start))) {
                start++;
            }

            while (start < end && 
                   !Character.isLetterOrDigit(s.charAt(end))) {
                end--;
            }

            if (Character.toLowerCase(s.charAt(start)) !=
                Character.toLowerCase(s.charAt(end))) {
                return false;
            }

            start++;
            end--;
        }

        return true;
    }
}
```


## Key Learning

- Always check pointer condition before accessing characters.
    
- Skip invalid characters before comparison.
    
- Use `start < end`, not `<=`, to avoid unnecessary middle comparison.
    


## Pattern Used: Two Pointer Inside Loop

Outer loop → iterate over words  
Inner loop → check palindrome using two pointers


# Your Logical Mistake

## Mistake: Declaring `isPalindrome = true` Outside the Loop

Incorrect structure:

```java
Boolean isPalindrome = true;

for(String word : words) {
    ...
}
```

### Why This Is Wrong

- Boolean state persists between iterations.
    
- If first word fails:
    
    ```
    isPalindrome = false
    ```
    
- Next word starts with `false` instead of `true`.
    
- Future words may never be evaluated correctly.
    

This causes incorrect results.


# Additional Common Mistakes in Palindrome Problems

1. Not breaking early after mismatch  
    → Leads to unnecessary comparisons.
    
2. Using `start <= end`  
    → Middle character does not require comparison.
    
3. Returning inside loop without full validation.
    
4. Forgetting case normalization in string problems.
    


# Mental Model

Palindrome check = Mirror comparison.

Left pointer moves right.  
Right pointer moves left.  
Mismatch means immediate termination.
