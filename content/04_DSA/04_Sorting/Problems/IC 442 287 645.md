## Problem

Given an array `nums` of size `n`  
where:

```java
1 ≤ nums[i] ≤ n
```

Some numbers appear twice.

Return all duplicate numbers.


# Your Complete Code

```java
class Solution {
    public List<Integer> findDuplicates(int[] nums) {
        search(nums);

        List <Integer> duplicate = new ArrayList<>();
        for(int i = 0;i<nums.length;i++){
            int count = nums[i]-1;
            if(i!=count && nums[i]==nums[count]){
                duplicate.add(nums[i]);
            }
        }
        return duplicate;
    }

    public int[] search(int[] nums){

        int i = 0;
        while(i<nums.length){
            int currect = nums[i]-1;
            if(currect>=0 && currect<nums.length && nums[i]!=nums[currect]){
                swap(nums,i,currect);
            }else{
                i++;
            }

        }
        return nums;
    }

    public void swap(int[] nums,int a,int b){
        int temp = nums[a];
        nums[a] = nums[b];
        nums[b] = temp;
    }
}
```


### Why This Condition Is Important

```java
nums[i] != nums[currect]
```

If duplicate exists:

Example:

nums = [2,2]

At index 1:

correct index of 2 = 1

But nums[1] is already 2.

So swapping would do nothing and cause infinite loop.

This condition prevents infinite swapping.


### After `search()` Completes

All numbers that can be placed correctly are placed.

Duplicates remain at wrong indices.


## Line 1

```java
int count = nums[i] - 1;
```

Since range is `1 to n`:

Correct index = value - 1.


## Line 3

```java
nums[i] == nums[count]
```

If correct position already contains the same number,  
then this is a duplicate.

Why?

Because:

- First occurrence is already at correct index.
    
- Second occurrence could not be placed.
    
- So it stayed somewhere else.
    

That is exactly what this condition detects.


# Full Logical Flow

1. `search()` rearranges array using cyclic sort.
    
2. Every number tries to go to its correct index.
    
3. If duplicate exists, one copy occupies correct index.
    
4. Other copy cannot be placed.
    
5. After sorting, mismatched indices reveal duplicates.
    


# Time and Space Complexity

Time Complexity: O(n)  
Each element swapped at most once.

Space Complexity: O(1)  
No extra array used.
